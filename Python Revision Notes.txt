Python Variables

- Python variables do not need to be declared with any particular types, and they can change type after they have been set
- Data type of a variable can be obtained using type() function
- Data type of a variable can be converted using casting 
- String variables can be declared either with single quotes or double quotes, multiline string variables can be declared using triple single or triple double quotes.
- Variable name rules:
	- Variable names are case-sensitive
	- Variable names must start with a letter or underscore 
	- Variable name can only containers alphanumeric and underscore	
	- Python keywords are reserved and cannot be used as variable names
	- Best practices are Camel Case (each word except the first word starts with capital letters. e.g., myFirstVariable), Pascal Case(each word starts with capital letters. e.g., MyFirstVariable), and Snake Case (each word is separated with underscore. e.g., my_first_variable)

Python can evaluate many types of values as True or False in an if statement. 0, empty strings (""), empty collections like list[], tuple(), set{}, etc are treated as False. Everything else is treated as True

If => to evaluate a condition 
if, elif => to evalute a condition in order from multiple mutually exclusive conditions
if, else/ if, elif, else => a fallback that executes when none of the preceding conditions are true

Short Hand if Syntaxs:
One-line if => if condition: value_if_true
One-line if, else => value_if_true if condition else: value_if_false
One-line multiple if, else => value_if_condition1 if condition1 else value_if_condition2 if condition2 else value_if_false

** A if statement cannot be empty. If it has no content, put in a pass statement to avoid error. The pass statement is a null operation, nothing happens when it executes. It serves as a placeholder. **

Python Match Statement => selects one of many code blocks to be executed. It is used for matching specific values/ patterns/ structure, or when there is too many elif

break => stop a loop even if the condition is met

Python Functions (def) => a block of code executes only when it is called. Functions needs to written only once, but can be used many times. It can take inputs(arguments), and return data as a result, which can be use directly in subsequent operations.

- Return statement => when a function reaches a return statement, it stops exection and send the result back

Parameters vs Arguments 
- Parameter is the variable listed inside the parentheses in function definitions
- Arguments is the actual value that is passed to the function when it is called 

When passing in arguments:

1. Number of arguments must match parameters
2. Default parameter values => if the function is called without an argument, the default value assigned to the parameter will be used
3. Keywords arguments (key=value syntax) => arguments are passed using parameter names, so argument order does not matter
4. Positional arguments => arguments are assigned based on their position, so they must be in the correct order
5. For mixing Positional and Keywords arguments => positional argument must come before keyword arguments

** to specify a function can have only positional arguments, add / after the argument in function definition
** to specify a function can have only keyword arguments, add * before the argument in function definition 
** to specify a function can have only mixed of positional and keyword arguments, add / after the positional argument and * before the keyword argument (positional argument must come before keyword arguments)

*args and **kwargs => allow functions to accept unknow number of arguments 

*args (Arbitrary Arguments) => allow a function to accept unknow number of positional arguments. These arguments are stored as tuple. 

**kwargs (Arbitrary Keyword Arguments) => allow a function to accept unknow number of keyword arguments. These arguments are stored as dictionary. 

Combining parameters, *args,**kwangs => the order must be:
1. regular parameters
2. *args
3. *kwargs

Unpacking arguments using * and ** operators
1. * operator => unpack a list, tuple, set into individual positional arguments(number of elements must match the function parameters)
2. ** operator => unpack a dictionary into individual keyword arguments(dictionary key must match to paramater names)

Python Scope => defines the visibility and lifetime of variables within different parts of a program
if, for and while statements do not create new scope. so variables created inside these statements belong to the global scope for outside functions, or local scope inside functions 

Functions, classes and module creates new scope, so variables defined inside them belong to their respective local scope. 

1. Local Scope => a variable created inside a function belongs to the local scope of that function and can only be used inside that function. 

2. Enclosing Scope => for nested functions, a variable defined in an outer function is accessible to inner functions, but a variable defined inside an inner function is not accessible to the outer functions 

3. Global Scope => a variable created outside any function (in the main body of the program) belongs to the global scope and can be used anywhere in the program. To make a change to or create a global variable inside a function, the global keyword must be used.

4. Built-in Scope => a variable belong to python, and are automatically available when the program runs.

nonlocal keyword => used in nested functions to modify a variable defined in the outer function.

Python uses LEGB rule to look up names (variables, functions and objects):
1. Local
2. Enclosing
3. Global Build-in 

Python Decorators => a function that takes another function as input and returns a new function as output. It allows modification and extension of the behaviour of a function without changing the function code.

# function inside a function/ nested function 
def get_multiple_func(n):
    print("executing get_multiple_func...")
    def multiple(x):
        print("executing multiple...")
        return n*x
    print("Type:", type(multiple)) # Type: <class 'function'>
    print("multiple function id:", id(multiple)) # multiple function id: 3024075856192
    print("closure:", multiple.__closure__) # closure: (<cell at 0x000002C018F5A500: int object at 0x00007FF8BAD549D8>,)
    print("free vars:", multiple.__code__.co_freevars) # free vars: ('n',)
    return multiple

# get_multiple_func is a function object in the global scope
# It is created when Python reads the def statement, but it is not executed yet
print("Type:", type(get_multiple_func)) # Type: <class 'function'>
print("get_multiple_func id:", id(get_multiple_func)) # get_multiple_func id: 3024075856352
print("closure:", get_multiple_func.__closure__) # closure: None
print("free vars:", get_multiple_func.__code__.co_freevars) # free vars: ()

print("Assign get_multiple_func to double...")
double = get_multiple_func(2) # get_multiple_func(n) => return multiple. get_multiple_func(2) executes and returns the multiple function object. double = multiple function object, with n = 2 stored in its closure.
print("Type:", type(double)) # Type: <class 'function'>
print("Address:", id(double)) # Address: 3024075856192
print("Closure:", double.__closure__) # Closure: (<cell at 0x000002C018F5A500: int object at 0x00007FF8BAD549D8>,)
print("Stored value:", double.__closure__[0].cell_contents) # Stored value: 2
print("Free vars:", double.__code__.co_freevars) # Free vars: ('n',)
print(double(3)) #6

# decorators must sit directly above the function definition
# passing a function as argument to changecase
# func is not a special keyword, it is a normal parameter name usually used to represents a function object
def changecase(func):
  print("original myfunction:", id(func))
  # myinner function, convert function output to upper cases 
  def myinner():
    return func().upper() # myinner return string object
  print("myinner: ", id(myinner))
  return myinner # return myinner function object

# changecase function object created, and myinner() code stored
print("changecase: ", id(changecase))

# myfunction = changecase(myfunction)
# changecase(func) => return myinner, myfunction = changecase(myfunction) => myfunction = myinner, where func = myfunction
@changecase
def myfunction():
  return "Hello Sally"

# myfunction function object created
print("changecase: ", id(changecase))
print("myfunction: ", id(myfunction))

print(myfunction()) # myfunction() = myinner(), where where func = myfunction

# decorator that takes in argument
def changecase(n):
  # decorator that takes in function 
  def changecase(func):
    # wrapper that coverts function output to lower/ upper cases 
    def myinner():
      if n == 1:
        a = func().lower()
      else:
        a = func().upper()
      return a 
    return myinner # return myinner function object
  return changecase # return inner changecase function object
# outer changecase function object created, inner changecase and myinner code stored

def style(case):
    def add_prefix(prefix):
        def changecase(func):
            def wrapper():
                text = func()
                if case.lower() == "upper":
                    text = text.upper()
                else:
                    text = text.lower()
                return prefix + text
            return wrapper
        return changecase
    return add_prefix

Lambda with built-in functions 
1. map (iterator) => applies a function to every item in an iterable 
2. filter (iterator) => create a list of filtered items for which a function returns True
3. sort => custom sorting using function as the key 

** An iterator is an object that produces values one at a time. It keeps track current item position, and is consumed as it is used. Once all item returned, iterator become exhausted and has no more value to return. An iterator can be consumed using next(), a for/ while loop, or by converting it to a collection such as list or tuple. **

Iterator object follows the iterator protocol consisting of __iter__() and __next__() methods

__iter__() method to return the iterator object itself
__next__() method to return the next value

Common examples of iterators include:
- map
- filter
- zip
- enumerate
- reversed
- generator
- iter

** An iterable is an object that stores a collection of values and can be looped over many times. e.g., list, tuple, set, strings, dictionaries. These iterable objects have a iter() method which takes in an iterable and returns an iterator. **

To create an object/ class as an iterator, implements both __iter__() and __next__() methods:

1. __iter__() => return self to return the iterator object itself
2. __next__() => return the next value in the sequence, and raise StopIteration when no more values

class MyNumbers:
  def __init__(self):
    self.a = 1
    # set starting value when object is created
  
  def __iter__(self):
    return self
    # return this same object as the iterator

  def __next__(self):
    if self.a <= 20:
      x = self.a
      self.a += 1
      return x 
      # return current object value, and increment self.a by 1
    else:
      raise StopIteration
      # stop the iteratorion
      # if StopIteration error is not raised, python return None in next line by default

Python map function

Nested function calls vs Nested function definitions

1. Nested function calls => calling one function inside another function call. e.g., doubled = list(map(lambda x: x * 2, numbers))
The inside call runs first:
   - create the lambda function object
   - call map(lambda, numbers)
   - map() returns a map object
   - list() consumes the map object, and create a list object

2. Nested function definitions => defining a function inside another functions
The outer function object is created first, and inner function objects are not created immediately. They are created only when outer function is called

Python recursive function

Recursion is when a function calls itself. Every recursive function must have 2 parts:
1. a base case - condition that stops the recursion 
2. a recursive case - function that calling itself with a modified argument

def countdown(n):
  # base case
  if n <= 0:
    print("Done!")
  else:
    print(n)
    # recursive case
    countdown(n - 1)

def find_max(numbers):
  # base case
  if len(numbers) == 1:
    return numbers[0]
  else:
    # recursive case
    max_of_rest = find_max(numbers[1:])
    return numbers[0] if numbers[0] > max_of_rest else max_of_rest

Python Object Oriented Principles

1. Inheritance => allows to define a class (child class/ derived class) that inherits all the attributes and methods from another class (parent class/ base class)
2. Polymorphism => allows same method/ function to be executed on different objects or classes, with each object/ class providing its own behaviour
3. Encapsulation => Protects object attributes and method in a class, and controls how they are accessed from outside the class.
4. Abstraction => hides internal implementation details from user and shows only the necessary attributes and methods.
5. Inner Class => class defined inside another class, the inner class can access the properties and methods of the outer class when outer class object is passed to it.

Inheritance:

1. Parent class/ base class => the class being inherited from. Any class can be a parent class, the syntax is the same as creating a normal class.
2. Child class/ derived class => the class that inherits the attributes and methods from another class. To create a child class, pass the parent class name as the parameter when defining the child class.

** Use the pass keyword when the child class has no attributes or methods of its own, the child class inherents the parent class attributes and methods **

** When __init__() is added to the child class, it is automatically called every time when the child class is used to create a new object. The child class __init__() overrides the parent class __init__() **

** To keep the inheritance of the parent class's attributes, add parent's __init__() function call in the child class's __init__(). 

** The child class inherits all parent class's methods unless the child class defines a method with the same name. If the child class defines a method with the same name, it overrides the parent method.

** To keep the inheritance of the parent class's attributes, add the super().__init__() function in the child class. The super().__init__() calls the parent __init__() in MRO (Method Resolution Order) so that child object can get the parent attributes. In multiple inheritance, super().__init__(*args) or super().__init__(**kwargs) can pass remaining arguments to the next class in the MRO for processing. **

Python Encapsulation:

Protecting an object's attributes and methods by controlling access from outside the class. It helps prevent accidental changes to attributes and hide internal details of how class works. 

1. Private attributes => they are created using a double underscore prefix (e.g., self.__age). Private attributes cannot be accessed directly from outside the class using object.attribute/ object.__attribute, usually getter, setter methods or property are used to read and update the value. However, they can still accessed indirectly using Python name mangling convention: object._Classname__attributes which is not recommended. 

2. Protected attributes => they are created using a single underscore prefix (e.g., self._salary). Protected attributes is for internal use within the class. They can still be accessed from outside the class, but by convention, accessing or modifying them directly outside the class should be avoided. 

Type of attributes:

Public => self.name => can be accessed directly using object.name
Protected => self._name => should avoid direct access using object._name from outside the class
Private => self.__name => cannot be accessed directly from outside the class using object.__name, it is accessed or modified through getter/setter methods or @property.

** Encapsulation does not mean the data can never be changed. It is mainly more for hiding internal data and controlling how it is accessed or modified. **

# Encapsultion 
class Person:
    def __init__(self, name, age): # attributes
        self.name = name
        self.__age = age # private attributes 

    def get_age(self): # getter method that get attribute value
        return self.__age
    
    def set_age(self, age): # setter method that set attribute value
        if age >= 0:
            self.__age = age
        else:
            raise ValueError("Age is invalid!")
        
p1 = Person("Hui", 33)
print(p1.get_age())
print(p1._Person__age) # mangle format for accessing private attribute
print(p1.name) # for accessing public attributes

p1.set_age(34)
print(p1.get_age())
print(p1._Person__age)

property() creates a property object that can contain a getter, setter, and deleter function. It allows controlled access to a private attribute using normal attribute syntax.

# Encapsultion 
class Person:
    def __init__(self, name, age): # attributes
        self.name = name # public attribute
        self.__age = 0   # private attribute
        self.age = age   # calls setter setter method 
    
    # age = property(original age getter function)
    # property() takes in the original getter function and returns a property object that can be accessed as an attribute
    @property
    def age(self): # getter method that allows to get attribute value
        return self.__age
    
    # age = age.setter(original age setter function)
    # age.setter() takes in the orignal setter function and returns a new property object  
    @age.setter
    def age(self, age): # setter method that set attribute value
        if age >= 0:
            self.__age = age
        else:
            raise ValueError("Age is invalid!")
        
p1 = Person("Hui", 33)
print(p1.age) # access getter method to read value
print(p1._Person__age) # mangle format for accessing private attribute
print(p1.name) # for accessing public attributes

p1.age = 34 # access setter method to update value
print(p1.age)
print(p1._Person__age)

1. Private methods => they are created using double underscores, and cannot be called/ accessed directly from outside the class using object.methodname()/ object.__methodname(). However, they can still be accessed indirectly using Python name mangling convention: object._Classname__methodname, but it is not recommended. 

2. Protected methods => they are created using single underscore for internal use within the class. Protected methods can still be accessed from outside the class using object._methodname(), but by convention, accessing or modifying them directly outside the class should be avoided.

** Guideline for using protect and private attributes/ methods **

- Use protected when the attribute/ method is meant for internal use, but child class maybe need to use and override it.
- Use private when the attribute/ method is meant be used only inside that class, to prevent accidental access or accidental overriding. 

# Encapsulation
class Calculator:
    def __init__(self): # attribute
        self.result = 0

    def __validate(self, number): # private method that cannot access from ouside the class
        if not isinstance(number, (int, float)): # checks if an object belongs to a certain class/type, isinstance(object, class/type)
            return False
        return True

    def add(self, number): # publib method 
        if self.__validate(number): # access private method
            self.result += number
        else:
            raise ValueError("Invalid number!")
    
    def sub(self, number): # public method 
        if self.__validate(number): # access private method
            self.result -= number
        else:
            raise ValueError("Invalid number!")

cal = Calculator()
cal.add(19)
print(cal.result)
cal.sub(4)
print(cal.result)
print(cal._Calculator__validate(20))

Python Abstraction 

Showing only what attributes/ methods the user needs to use, while hiding how it works internally inside the class.

1. Using normal public, protected or private attributes and methods
2. Using built-in ABC class and @abstractmethod => A formal abstract class is usually created by inheriting from the built-in ABC class. It can contain:
- Normal method => methods can be inherited and used directly by child classes
- @abstractmethod => required methods that must be implemented or overridden by child classes before objects of those child classes can be created. Objects cannot be created directly from the parent abstract class, it can only be created from child classes after all required abstract methods are implemented.

# Stack class
class Stack:
    def __init__(self):
        self.head = None # head node =>None
        self.size = 0 # stack size
    
    def push(self, value): # push method
        new_node = Node(value) # create a new node, new_node=>A, A.next=>None
        if self.head: # if stack is not empty, self.head=>X, X.next=>None 
            new_node.next = self.head # new_node.next=head, new_node=>A, A.next=>X, X.next=>None 
        self.head = new_node # If stack is empty, self.head=>A, A.next=>None. If stack is not empty, self.head=>A, A.next=> X, X.next=>None 
        self.size += 1 # increase stack size by 1
    
    def pop(self): # pop method
        if self.isEmpty(): # if stack is empty 
            return "Stack is empty"
        popped_node = self.head
        self.head = self.head.next # self.head=>A, A.next=> X, X.next=>None, self.head = self.head.next, self.head=> X, X.next=>None
        self.size -= 1 # reduce stack size by 1
        return popped_node.value

    def peek(self): # peek method
        if self.isEmpty():
            return "Stack is empty"
        return self.head.value

    def isEmpty(self): # is empty method
        return self.size == 0
    
    def printAllNodes(self):
        currentNode = self.head
        while currentNode:
            print(f"{currentNode.value} =>", end=" ")
            currentNode = currentNode.next
        print("None")

class Queue:
    def __init__(self):
        self.head_node = None
        self.rear_node = None
        self.size = 0

    def enqueue(self, value): # enqueue method
        new_node = Node(value) # create a new node 
        if self.rear_node: # if queue is not empty
            # head_node|rear_node=>A.value, A.next=>None. So self.rear_node.next = new_node => A.next = new_node
            self.rear_node.next = new_node # head_node|rear_node=>A.value, A.next=>new_node.value, new_node.next=> None
            self.rear_node = new_node # head_node=>A.value, A.next=>(rear_node=>)new_node.value, new_node.next=> None
        else:
            self.head_node = self.rear_node = new_node # if queue is empty head_node|rear_node=>new_node.value, new_node.next=>None
        self.size +=1 # increase queue size by 1
        return self.head_node
    
    def dequeue(self): # dequeue method 
        if self.isEmpty():
            return "Queue is empty"
        temp_node = self.head_node
        self.head_node = self.head_node.next # move head pointer to next element in the queue
        self.size -= 1 # decrease queue size by 1 

        if self.head_node is None: # reset rear_node if queue is empty, 
            self.rear_node = None

        temp_node.next = None # disconnect temp_node from rest of the queue 
        return temp_node.value  

    def peek(self): # peek method 
        if self.isEmpty():
            return "Queue is empty"
        return self.head_node.value

    def isEmpty(self):
        return self.size == 0
    
    def printQueue(self):
        if self.isEmpty():
            return "Queue is empty"
        temp_node = self.head_node
        while temp_node:
            print(f"{temp_node.value} =>", end=" ")
            temp_node = temp_node.next
        print("None")

Hash table => data structure that uses keys to insert, find, and delete values quickly. The hash function converts a key into an index which tells a hash table which bucket should be used.. A hash table contains buckets, which are storage locations that store values. 

Hash table collision => when hash function return same index for different keys

1. Chaining => each bucket can store multiple values by using list or linked list
2. Open addressing => if the target bucket is occupied, find another bucket. Linear probing => check the next bucket one by one which can create clustering. Quadratic probing => check the next bucket using square numbers. Double hashing => use a second hash function to decide the next bucket

Using class as hash table

class HashTable:
    def __init__(self):
        self._bucket = 5
        self.table = [[] for _ in range (self._bucket)]

    def hash_function(self, key):
        idx = 0
        for char in key:
            idx += ord(char)
        return idx % self._bucket

    def add(self, value):
        index = self.hash_function(value)
        self.table[index].append(value)

    def find(self, value):
        index = self.hash_function(value)
        return value in self.table[index]

    def delete(self, value):
        index = self.hash_function(value)
        if value in self.table[index]:
            self.table[index].remove(value)

Tree => data structure that is similar to Linked List such that each node contains data and the reference to other node. In a tree, a single node can have multiple next node. 
1. Binary Tree => each node has maximum 2 child nodes, left child node and right child node
2. Binary Search Tree (BST) => for every node, all values in the left subtree are smaller than the node's value, and all values in the right subtree are greater than the node's value. 
3. AVL Tree => BST that is self-balancing so that for every node, the height difference between the left and right subtrees is at most one. 

List VS linked list VS Tree

- List are fast for accessing an element at a specific index, but inserting and deleting elements may require other elements to shift in memory 
- Linked List are fast for inserting and deleting elements when head node is known, but accessing an element needs traversing the nodes one by one.
- Tree are fast for accessing, inserting and deleting the nodes.

Binary tree traversal => Go through a tree by visiting a node at a time for all nodes

1. Breadth First Search (BFS) => nodes at the same level are visited before next level in the tree
2. Depth First Search (DFS) => visit nodes from all the way down the tree, branch by branch
   - pre-order, root => left subtree => right subtree 
   - in-order, left subtree => root => right subtree 
   - post-order, left subtree => right subtree => root

def height(node):
    if node is None:
        return 0
    # print(node.value)
    return 1 + max(height(node.left_node), height(node.right_node))

def nodesAtLevel(node, level):
    if node is None:
        return
    
    if level == 0:
        print(f"{node.value}")
        return
    #print(f"{node.value} go to left_node")
    nodesAtLevel(node.left_node, level-1)
    #print(f"{node.value} go to right_node")
    nodesAtLevel(node.right_node, level-1)

def breadthFirstSearch(node):
    tree_height = height(node)
    for level in range(tree_height):
        print (f"level: {level}")
        nodesAtLevel(node, level)

def minBST(node):
    # if it is a empty tree
    if node is None:
        return None
    
    while node.left_node:
        node = node.left_node
    return node

def maxBST(node):
    # if it is a empty tree
    if node is None:
        return None
    while node.right_node:
        node = node.right_node
    return node

def insert(node, value):
    if node is None:
        return TreeNode(value)
    elif value < node.value: # if value less than node value, insert left subtree
        node.left_node = insert(node.left_node, value)
    elif value > node.value:
        node.right_node = insert(node.right_node, value) # if value more than node value, insert right subtree
    else:
        print("Node already exist") # if node already exist
        return 
    
    # print("return node...")
    return node
